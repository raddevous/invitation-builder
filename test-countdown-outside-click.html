✅ FIXED: Countdown Settings Outside Click/Scroll Handlers

Problem Solved: The unsaved changes dialog now triggers when clicking outside or scrolling outside the Countdown Settings panel!

What Was Fixed:

1. Click Outside Handler

useEffect(() => { if (!showCountdownSettingsPanel) return; const handleClickOutside = (e: MouseEvent) => { const panel = document.querySelector('[data-countdown-settings-panel="true"]'); if (panel && !panel.contains(e.target as Node)) { handleCloseCountdownSettingsPanel(); } }; document.addEventListener('mousedown', handleClickOutside); return () => document.removeEventListener('mousedown', handleClickOutside); }, [showCountdownSettingsPanel, hasUnsavedCountdownChanges, handleCloseCountdownSettingsPanel]);

2. Scroll Outside Handler

const handleScrollOutside = (e: Event) => { const panel = document.querySelector('[data-countdown-settings-panel="true"]'); if (panel && !panel.contains(e.target as Node)) { handleCloseCountdownSettingsPanel(); } }; document.addEventListener('scroll', handleScrollOutside, true); window.addEventListener('scroll', handleScrollOutside, true);

3. Panel Data Attribute

<div data-countdown-settings-panel="true" className="fixed z-50 shadow-2xl flex flex-col..." >

How It Works Now:

  1. Open Countdown Settings panel - Click on the countdown section in edit mode
  2. Make changes - Modify colors, text, or any settings
  3. Try to close:

    • ✅ Scroll outside panel → Shows unsaved dialog
  4. Dialog options:
    • Apply → Saves changes and closes panel
    • Discard → Abandons changes and closes panel
