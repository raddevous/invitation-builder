Countdown Section Unsaved Dialog Test

This test verifies that the unsaved changes dialog works correctly in the Countdown Settings panel.

Test Steps:

  1. Open the Countdown Settings panel
  2. Make a change (e.g., modify heading text, change colors, etc.)
  3. Try to close the panel without applying changes
  4. Verify that the unsaved changes dialog appears
  5. Test both "Apply" and "Discard" options

Expected Behavior:

  • Dialog should appear when there are unsaved changes
  • Dialog should NOT appear when there are no unsaved changes
  • "Apply" should save changes and close panel
  • "Discard" should abandon changes and close panel
Current Status: The code has been updated to use separate state variables for countdown changes:
  • pendingCountdownChanges instead of pendingChanges
  • hasUnsavedCountdownChanges instead of hasUnsavedChanges
  • handleCountdownChange instead of handleChange

Debug Information:

The following functions have been implemented:

  • handleCloseCountdownSettingsPanel() - Checks for unsaved changes and shows dialog
  • handleCountdownChange() - Updates pending changes and sets unsaved flag
  • handleApplyChanges() - Applies changes and closes panel
  • handleConfirmApply() - Dialog handler for applying changes
  • handleConfirmDiscard() - Dialog handler for discarding changes
